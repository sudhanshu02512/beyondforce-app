Composition

Composition means building a big component using small components.

It consist of two concepts

  1. Parent & Child Components (Parent Composition)
  2. Slots (Child Composition)

Parent & Child Component (Parent Composition)

One component can compose another component, this is also known as Parent-Child concepts in LWC.

Example:

//parent.html

<template>

    <c-header></c-header>

    <c-account-list></c-account-list>

    <c-footer></c-footer>

</template>

  • Here you can see the parent component is composed of 3 child components.
  • Infact Parent and child component concept in LWC is a part of composition.

The code from the child component is inserted into the parent component on runtime using shadow dom.

Slots(Child Composition)

A slot allows a parent component to pass markup (HTML content) into a child component.

When you call any child component from parent, if you pass any as html as composition then that html will not be rendered on UI and will be discarded.

//parent.html

<template>

    <c-child-component>

        <div>This is heading</div> → Discard at runtime

    </c-child-component>

</template>

Here we have written a div inside the child component calling but that div code will be discarded when the child component is loading.

But if we have declared a slot in the child component then this div will be inserted into the child component from the parent.

Example

//parent.html

<template>

    <c-child-component>

        <div>This is heading</div>

    </c-child-component>

</template>

//child.html

<template>

         <slot></slot>

</template>

Output:

<template>

    <c-child-component>

<slot>

        <div>This is heading</div>

</slot>

    </c-child-component>

</template>

Types of slots

There are two types of slots

  1. Default Slot
  2. Named Slots
