Apex

Apex is a salesforce service side language that enables developers to add business logic to DML operations, API calls, and act as controller for UI frameworks such as VF Pages, Aura and LWC.

Features of apex

  • It is a non declarative tool of automation that runs natively on Salesforce.
  • It is a strongly typed language that means variables should be defined with data types.
  • It is not a case-sensitive language.
  • It supports Object Oriented Programming (OOPs).
  • It is tightly integrated with salesforce metadata meaning it understands salesforce objects and fields as well as SOQL and SOSL directly.
  • It supports OLS, FLS and RLS security enforcement.
  • It is one-time compiled and whenever it is called then already compiled code is executed that's why it behaves as interpreted language.
  • It has two ways of execution: one is synchronous and other is asynchronous.
  • It supports API callouts.
  • It can behave as a web service.
  • It has its own testing framework known as Apex Testing Framework.

From where you can call or execute apex?

You can call apex from the below things.

  • Apex Triggers
  • Lightning Web Component s
  • Aura Components
  • Flows
  • VF Pages

When should I use apex?

  • Use apex when declarative tools like flows, validation rules, email alerts, etc. cannot solve the business requirement.
  • Requirements involve complex DML operations
  • The requirement involves bulk data processing.
  • Calling external APIs and then performing complex operations.
  • Working with salesforce data from LWC and Aura.
