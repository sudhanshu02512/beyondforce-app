Future Methods

Future methods are special types of methods that are always run asynchronously.

  • It is first queued then executed when resources are available.
  • Future Method transaction have different governor limit as compared to normal apex transaction for eg. you can query 200 SOQLs in future transactions while in normal transactions only 100 are allowed.

Syntax

@Future | @Future(Annotation_Parametes)

 public static void futureMethodName(InputParam parameters) {

    // Your future method logic here   

 }

Considerations

  • Future methods must be declared as static
  • It can only return void type.
  • You can only pass primitive data type or collection of primitive data type as an input parameter, sObjects or Objects are not supported

Limitations

  • Sobject and object parameters are not supported.
  • You cannot get a job id inside the apex of future method transaction, can only get from ‘Apex Jobs’ in setup.
  • The future method cannot be called from a future or batch method.
  • You can only enqueue one queueable job per future method transaction.
  • You cannot call a batch class from a future method but can schedule schedulable class.

Future Annotation Parameters

Future methods have only one annotation parameter that is callout.

@Future(callout=true)

  • It allows future methods to call external API.
  • If your future is calling external api and you have not added an annotation parameter then it will give you an exception ‘System.CalloutException: Callout not allowed from this future method. Please enable callout by annotating the future method. eg: @Future(callout=true)’.

Sample Code

@Future(callout=true)

    public static void futureMethodWithCallout() {

        String pincode = '201013'; // Example pincode

        String endpoint = 'https://api.postalpincode.in/pincode/';

        endpoint += pincode;

        HttpRequest req = new HttpRequest();

        req.setEndpoint(endpoint);

        req.setMethod('GET');

        Http http = new Http();

        HttpResponse res = http.send(req);

        if(res.getStatusCode() == 200) {

            System.debug(res.getBody());

        } else {

            System.debug(res.getStatusCode());

        }

    }

Use Cases

  • Calling out api from the trigger as directly calling from trigger or trigger handler gives you an exception ‘System.CalloutException: Callout from triggers are currently not supported.’
  • To prevent MIXED_DML_OPERATION Exception happens when you try to perform DML on Setup objects and Non-Setup objects in the same transaction.
  • To move any logic to separate transactions.

Monitor Job

You can monitor future method jobs in Apex Jobs under Setup.

  • It is available as Job Type = Future

Job Chaining

Chaining with another future method is not allowed but you can chain other asynchronous features like Queueable Class.

Queueable Class Job Chaining

But you can enqueue only 1 queueable job per future transaction.

Sample Code

@Future

    public static void futureMethodJobChaining() {

        QueueableManager qm = new QueueableManager();

        System.enqueueJob(qm);

    }

Batch Class Job Chaining

You cannot batch class from a future method as Database.executeBatch cannot be called from a batch start, batch execute, or future method.

Scheduleable Class Job Chaining

You can schedule any schedulable class from a future method.

@Future

    public static void futureMethodJobChaining() {

        ScheduleableManager sm = new ScheduleableManager();

        String cronExp = '0 0 12 15 4 ? 2026';

        System.schedule('Scheduleable Job', cronExp, sm);

        ScheduleableManager sm2 = new ScheduleableManager();

        String cronExp2 = '0 0 12 15 4 ? 2026';

        System.schedule('Scheduleable Job2', cronExp2, sm2);

    }
