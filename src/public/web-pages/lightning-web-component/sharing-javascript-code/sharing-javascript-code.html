Sharing JavaScript Code

There are two cases in which we have to share code.

  1. Inside the same component using additional javascript.
  1. Additional Javascript(Utility Javascript)
  1. Between two different component using one common component(Utility Component)

Other Cases are third party js, etc.

To share code we are using export and import concept of javascript

Additional Javascript (Utility JavaScript File)

Exporting & importing default variables and functions

Exporting

        defaultUtility1.js

function defaultFunction(){

    console.log('defaultFunction');

}

export {defaultFunction as default};

OR

export default defaultFunction;

Importing

shareJavaScript.js

import defaultFunction from './defaultUtility1';

export default class ShareJavaScript extends LightningElement {

    constructor(){

        super();

        defaultFunction();

    }

}

Same logic we have for variables also.

Naming collision

In case while importing the name of a reference variable or function is colliding with the existing name of another variable or function then you can give any name while importing.

Example

import defaultFunction1 from './defaultUtility1';

OR

import fun2 from './defaultUtility1';

In fact you can give any name as per your wish.

Exporting & importing named variables and functions

Export

namedUtility2

let phone = '1232123';

function add(x,y){

    return x+y;

}

export {phone,add};

Import

import {phone,add} from './namedUtility2';

Name Collision

You can give alias of variable or function while importing

Example

import {phone as myPhone,add as addFunction} from './namedUtility2';

Exporting both named and default at the same time

//defaultNamedUtility2

let name = 'Test';

function getInfo(){

    console.log('getting info');

}

export {name, getInfo as default, getInfo};

Import

import {name as name1,getInfo} from './defaultNamedUtility2';

import getInfo2 from './defaultNamedUtility2';

Utility Component

Code stored in the utility component but shared in other components using export and import.

utilityComponent.js

Export

export let name = 'abc';

Import in other component Js

import {name as name2} from 'c/utilityComponent';

Third Party Javascript

You can also use javascript written by third party in lwc.

  1. Store the JS in Static Resource
  2. Call it using the lightning/platformResourceLoader module.
  1. loadScript - For third Party JS
  2. loadStyle - For third Party CSS
  1. Use Promise.all([])

Syntax

import LEAFLET from '@salesforce/resourceUrl/leaflet';

import { loadScript, loadStyle } from 'lightning/platformResourceLoader';

import getAccounts from '@salesforce/apex/LeafletMapController.getAccounts';

renderedCallback() {

        if (this.leafletInitialized) {

            return; // Prevent multiple loading

        }

        this.leafletInitialized = true;

        Promise.all([

            loadScript(this, LEAFLET + '/leaflet.js'),

            loadStyle(this, LEAFLET + '/leaflet.css'),

            getAccounts()

        ])

        .then((res) => {

            this.accountData = res[2];

            this.initializeMap();

            console.log(res[2]);

        })

        .catch(error => {

            console.error(error);

        });

    }

Tips:

  1. Always upload as Static Resource
  2. Use loadScript / loadStyle
  3. Access DOM inside renderedCallback()
  4. Avoid direct document.querySelector()
